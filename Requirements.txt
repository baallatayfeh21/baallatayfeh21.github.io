	Hosting Your Website on GitHub [Video- How to Host a Website On Github Pages]
o	Create a GitHub Account: If you don't already have one, sign up for a GitHub account at github.com.
o	Create a New Repository:
	Log in to your GitHub account.
	Click on the "+" sign in the top right corner and select "New repository."
	Name your repository <your-username>. github.io. For example, if your username is student123, name your repository student123.github.io.
	Choose whether you want your repository to be public or private.
	Click on "Create repository."
o	Upload Your Website Files:
	Upload your website files to the repository you just created. Ensure that your main HTML file is named index.html.
	You can drag and drop files directly into the GitHub web interface or use Git commands to push files to your repository.
o	Enable GitHub Pages:
	Go to your repository's settings.
	Scroll down to the "GitHub Pages" section.
	Under "Source," select "master branch."
	Click on "Save."
o	Access Your Website:
	After enabling GitHub Pages, GitHub will provide you with a URL where your website is hosted. It will typically be https://<your-username>.github.io.
	Open a web browser and navigate to this URL to view your hosted website.


Tip: The way a Web server chooses a default page to display is simple: When it receives a request that does not include a file name, it consults a list of default file names (such as "index.html", "index.htm", and so forth) and displays the first matching file it finds.
